# 数据库和数据库系统 (Database and Database Systems)

# 数据库 (Database)

A collection of data, typically describing the activities of one or more related organizations.
数据库是数据的集合，用于描述一个或多个相关组织的活动。

数据库包括实体和实体间的联系

  • 实体 (Entities)

学生、教工、课程，教室

  • 实体间的联系 (Relationships between entities)

学生登记的课程、教师教授的课程，用于授课的教室

# 数据库管理系统 (Database Management System)

A DBMS is a collection of software programs to enable users to create, maintain and utilize a database.
数据库管理系统是辅助用户管理的利用大数据集的软件。

# 数据模型 (Data Models)

在设计过程中要避免使用像 age 这样的字段，因为它的值是不断变化的，而应该使用 DOB (Date of birth)，age 可以根据 DOB 计算到。以后的例子中使用 age，因为可读性好。

# 关系模型 (Relational Model)

关系模型的核心数据描述结构是关系 (relation)，它可以被认为是记录 (record) 的集合。

基于数据模型的数据描述称为模式 (schema)。

在关系模型中，一个关系的模式需要说明这个关系的名字、每个字段（也称为属性或列）的名字及每个字段的类型。例如：

Students(sid:string, name:string, login:string, age:integer, gpa:real)
sidnameloginagegpa
10000Alicealice@a183.4
10001Bobbob@b193.2
10002Charliecharlie@c203.8

Students 关系中的每一行描述了一条学生记录。这个描述是不全面的 —— 例如，没有包括学生的身高。

完整性约束是关系中的记录必须满足的条件，通过说明完整性约束，可以使学生集合的描述更精确。

# 数据库管理系统的抽象级别

数据库描述有如下三级抽象模式组成：[概念模式]、物理模式和外模式。

数据定义语言 (DDL) 用于定义外模式和概念模式。

  • 概念模式
    概念模式（有时也称为逻辑模式）以 DBMS 数据模型的形式描述储存的数据。
    在关系型 DBMS 中，概念模式描述储存在数据库中的所有关系，这些关系包括实体信息和联系信息。
Students(sid:string, name:string, login:string, age:integer, gpa:real)
Faculty(fid:string, fname:string, sal:real)
Course(cid:string, cname:string, credits:integer)
Rooms(rno:integer, address:string, capacity:string)

Enrolled(sid:string, cid:string, gtade:string)
Teaches(fid:string, cid:string)
Meets_IN(cid:string, rno:integer, time:string)
  • 物理模式
    物理模式描述储存细节。本质上看，物理模式描述了概念模式中关系在磁盘和磁带等二级储存设备上实际是如何储存的。

首先必须决定使用什么文件结构来储存关系，并创建称为索引的辅助数据结构，以加速数据检索操作。

· 使用为排序的记录文件存储所有关系
· 分别在 Students、Faculty 和 Courses 关系的第一列，Faculty 的 sal 列，以及 Rooms 的 capacity 列上创建索引。
  • 外模式
    外模式常常也称之为 DBMS 的数据模型，允许为单个用户和用户组定制（和授权）数据存储。

任何给定的数据库只有一个概念模式和一个物理模式，但可以有多个外模式

每一个外模式由一个或多个视图和来自概念模式的关系组成。

# 允许学生查找授课老师的名字和课程登记的视图
Courseinfo(cid:string, fname:string, enrollment:integer)

用户可以像关系一样使用视图，查询视图中的记录。

在概念模式中并不包括 Courseinfo，是因为它能从概念模式中的关系计算出来，额外的存储是冗余的。

# 数据独立性

使用 DBMS 的一个重要的有点是它能够提供数据独立性，即当数据的结构和存储方式发生变化时，应用程序不受影响。

# Faculty 关系可以被下面两个关系替代
Faculty_public(fid:string, fname:string, office:integer)
Faculty_private(fid:string, sal:real)

# Database Languages

  • Data Definition Language (DDL)

    • A language that specifies data schemas

  • Data Manipulation Language (DML)

    • A language to facilitate the retrieval, update of data in the database

For retrieval, we query the database with the query language, which is part of the DML

# Database Design

  1. Requirement Analysis
  • What data is to be stored?
  • What applications must be built.
  • What operations are the most frequent and subject to performance requirement
  1. Conceptual database design
  • High-level description of
    • Data to be stored
    • Constraints
  • Often carried out using the ER model
  1. Logical database design
  • Choose a DBMS to implement our database design.
  • Convert the conceptual database design into a database schema in the data model of the chosen DBMS.
    • Convert ER schema into a relational database schema.
  1. Schema Refinement
  • Analyze the collection of relations in our relational
    database.
  • Identify the potential problems.
  • Refine the schema
  1. Physical database design
  • Ensure that the design meets the performance
    requirement.
  • Build index, clustering some tables etc.
  • Redesign parts of the database schema.
  1. Application and security design
  • Write application programs.
  • Identify data that can be accessible by certain types of
    users.
  • Take steps to ensure that access rules are enforced.
