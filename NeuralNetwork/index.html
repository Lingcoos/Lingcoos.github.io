# 神经网络如何工作的

# 预测机

输入 X → 计算 → 输出 Y

  • 线性预测机的训练
    X、Y 呈线性关系，即 Y = CX，推测出 C 值：
  1. 设定任意 C ，输出 Y
  2. 将 Y 与真实值比较
  3. 计算 Y 的误差值（误差值 = 真实值 - 计算值）
  4. 根据比较结果调整 C 的值
  5. 反复迭代

我们不能精确的知道一些事情是如何运作的，但我们可以尝试用模型来估计其运作方式。在模型中，我们可以调整参数来不断趋近结果。

# 分类器

通过调整斜率来对不同的事物进行分类

  • 线性分类器的训练
    线性分界线，即 y=Axy = Axy=Ax (参数 A 控制斜率)
  1. 设定任意 A
  2. 将 y 与期望值比较
  3. 计算 y 的误差值 E（误差值 = 期望目标值 - 实际输出值）
  4. 计算斜率调整值 ΔA (ΔA=E/xΔA = E/xΔA=E/x)
  5. 得出分界线 y=(A+ΔA)xy = (A + ΔA)xy=(A+ΔA)x
  6. 计算其他样本，更新分界线

发现无论怎么计算，分界线都偏向某一单一样本，无法整齐划分。改进方法，引入调节系数 LLL。

  • 学习率 (learning rate)
    ΔA=L(E/x)\Delta A = L (E/x)ΔA=L(E/x)
    我们重复先前的操作，给定学习率为 0.5 ，即每次调节一半。
  1. 给定初始值 A
  2. 算出 y 并与期望值比较
  3. 计算误差值 E
  4. 计算斜率调整值 ΔA=L(E/x)\Delta A = L (E/x)ΔA=L(E/x)
  5. 更新后的 A 值为 A=A+ΔAA = A + \Delta AA=A+ΔA
  6. 计算其他样本，更新分界线

真实世界的样本可能充满噪声或错误信息，学习率（适度更新）有助于限制错误样本的影响。

# 神经元

虽然一些计算机的运算速度比一般生物的大脑要快很多，但是即使是杂鱼的大脑，其能力也远比计算机强。
计算机按照严格的串行顺序，准确具体地处理数据，不存在模糊性和不确定性。而另一方面，动物的大脑表面上看起来慢得多，缺似乎以并行方式处理信号，模糊性是其计算的一直特征。

  • 神经元

    虽然神经元有各种形式，但是所有的神经元都是将电信号从一端传输到另一端，从一个神经元传到另一个神经元。这就是身体感知外界信号的机制。
    我们需要多少个神经元才能执行相对复杂的有趣任务呢？
    能力非常强的人类大脑大约有 1000 亿个神经元！
    一只果蝇有约 10 万个神经元。10 万个神经元，恰好是现代计算机试图复制的范围。
    一只线虫仅仅具有 302 个神经元。与今天的计算机资源相比，简直是微乎其微！但是，一只线虫能够完成一些大多数传统计算机都难以完成的任务。

  • 神经元的工作方式
    神经元接受了一个电输入，输出另一个电信号。这看似与分类器或预测器一模一样。输入→处理→输出。
    所以，我们可以将神经元表示为线性函数吗？当然可以，但是不可以这样做。生物的神经元与简单的线性函数不一样，不能简单地对输入做出的响应，生成输出。也就是说，输出不能采用这种形式：
    输出 = (常数 ×\times× 输入) + 也许另一常数

观察表明，神经元并不会立刻反应，而是会抑制输入，直到输入增强，强大到可以触发输出。也就是说，在输出之前，输入要达到一个阈值。直观上，神经元不希望处理微小的噪声信号，而只是传递有意识的明显信号。
在数学上，有许多激活函数可以达到这样的效果。例如一个简单的阶跃函数。

你可以看到在输入值较小的情况下，输出为零。然而，一旦输入达到阈值，输出就一跃而起。具有这种行为的人工神经元就像一个真正的生物神经元。输入达到阈值时，神经元就激发了。
我们可以改进阶跃函数。下图所示的 S 形函数称为 S 函数。这个函数更自然，更接近现实。自然界很少有冰冷尖锐的边缘！

我们将继续使用这种平滑的 S 函数来制作神经网络。虽然人工智能研究人员还使用其他外形类似的函数，但是 S 函数简单，并且事实上非常常见，因此 S 函数对我们非常重要。
S 函数，有时也称为逻辑函数：

y=11+e−x\Large y=\frac{1}{1+e^{-x}}y=1+e−x1​

  • 构建人工神经
    神经元可以接受多个输入。对于这些输入我们需要做什么呢？
    我们只需要对它们进行相加，得到最终总和，作为 S 函数的输入，然后输出结果。
    这实际上就反映了神经元的工作机制。

如果组合信号不够强大，那么 S 阈值函数的效果是抑制输出信号。
我们用以下方式来连接若干的神经元。

如果一个神经元被激发了，它同时提供信号给更多的神经元。
将这种自然形式复制到人造模型的一种方法是，构造多层神经元，每一层的神经元都与在其前后的神经元相互连接。

三层是神经元，每一层有三个人工神经元或节点。每个节点都与前一层或后一层的其他每一个节点互相连接。
太酷啦！那能不能调整节点之间的连接强度呢？当然可以！我们可以调整输入总和或是 S 阈值函数的形状，但是，调整 S 阈值函数的形状要复杂得多。
所以，我们前者。通过改变权重的大小来调整连接强度。

这种一致性的完全连接结构事实上可以相对容易的编码成计算机指令。神经网络的学习过程将会弱化这些实际上不需要的连接（也就是这些连接的权重趋近与 0，零权重意味着节点断开）。

比起现代计算机，杂鱼脑袋空空，但是也可以进行复杂的任务。

比起计算机，大脑对于损坏和不完善的信号有难以置信的弹性。

互相连接的神经元的大脑是组成人工神经网络的灵感来源。

# 矩阵乘法大有用途

矩阵有着让人闻风丧胆的声誉。它们唤起了我们的记忆，让我们记起在学校进行矩阵乘法时，那种让人咬牙切齿、枯燥费力的工作，以及那毫无意义的时间一小时一小时地流逝。

那么，矩阵如何帮助我们简化计算呢？

首先，矩阵允许我们压缩所有这些计算，把它们变成一种非常简单的缩写形式。由于人类不擅长于做大量枯燥的工作，而且也很容易出错，因此矩阵对人类帮助很大。

其次，许多计算机编程语言理解如何与矩阵一起工作，计算机编程语言能够认识到实际的工作是重复性的，因此能够高效高速地进行计算。

总之，矩阵允许我们简洁、方便地表示我们所需的工作，同时计算机可以高效地完成计算。

尽管我们在学校学习矩阵时有一段痛苦的经历，但是现在大家知道我们为什么要使用矩阵了吧。让我们开始使用矩阵，揭开矩阵的神秘面纱。

# 学习来自多个节点的权重

先前，我们通过调整节点线性函数的斜率参数，来调整简单的线性分类器。我们使用误差值，也就是节点生成了答案与所知正确答案之间的差值，引导我们进行调整。实践证明，误差与所必须进行的斜率调整量之间的关系非常简单，调整过程非常任意。

当输出和误差是多个节点共同作用的结果时，我们如何更新连接权重呢？下图详细阐释了这个问题。

当只有一个节点前馈信号到输出节点，事情要简单得多。如果有两个节点，我们如何使用输出误差值呢？

使用所有的误差值，只对一个权重进行更新，这种做法忽略了其他连接及其权重，毫无意义。多条连接都对这个误差值有影响。

只有一条连接造成了误差，这种机会微乎其微。如果我们改变了已经 “正确” 的权重而使情况变得更糟，那么在下一次迭代中，这个权重就会得到改进，因此神经网络并没有失去什么。

一种思想就是在所有造成误差的节点中平分误差，如下图所示。

另一种思想是不等分误差。与前一种思想相反，我们为较大连接权重的连接分配更多的误差。为什么这么做呢？这是因为这些连接对造成误差的贡献较大。下图详细阐释了这种思想。

此外，有两个节点对输出节点的信号做出了贡献，它们的连接权重分别是 3.0 和 1.0 。如果按权重比例分割误差，那么我们就可以观察到输出误差的 3/4 应该可以用于更新第一个较大的权重，误差的 1/4 可以用来更新较小的权重。

我们可以将同样的思想扩展到多个节点。如果我们拥有 100 个节点连接到输出节点，那么我们要在这 100 条连接之间，按照每条连接对误差所作贡献的比例（有连接权重的大小表示），分割误差。

你可以观察到，我们在两件事情上使用了权重。第一件事情，在神经网络中，我们使用权重，将信号从输入向前传播到输出层。此前，我们就是在大量地做这个工作。第二件事情，我们使用权重，将误差从输出向后传播到网络中。我们称这种方法为反向传播，你应该不会对此感到惊讶吧。

如果输出层有 2 个节点，那么我们对第二个输出节点也会做同样的事情。第二个输出节点也会有误差，这个误差也可以通过相连的连接进行类似的分割。

让我们详细解释一下这是什么意思。想象一下，一个非常复杂、有波峰波谷的地形以及连绵的群山峻岭。在黑暗中，伸手不见五指。你知道你是在一个山坡上，你需要到坡底。对于整个地形，你没有精确的地图，只有一把手电筒。你能做什么呢？你可能会使用手电筒，做近距离的观察。你不能使用手电筒看得更远，无论如何，你肯定看不到整个地形。你可以看到某一块土地看起来是下坡，于是你就小步地往这个方向走。通过这种方式，你不需要完整的地图，也不需要事先制定路线，你一步一个脚印，缓慢地前进，慢慢地下山。

在数学上，这种方法称为梯度下降，你可以明白这是为什么吧。在你迈出一步之后，再次观察周围的地形，看看你下一步往哪个方向走，才能接近目标，然后，你就往那个方向走出一步。你一直保持这种方式，直到非常欣喜地到达了山地。梯度是指底面的坡度。你走的方向是最陡的坡度向下的方向。

这种酷炫的梯度下降法与神经网络之间有什么联系呢？好吧，如果我们将复杂困难的函数当作网络误差，那么下山找到最小值就意味着最小化误差。这样我们就可以改进网路输出。这就是我们希望做到的！

我们要改变步子大小，避免超调，这样就会避免在最小值的地方来回反弹，这是一个必要的优化。你可以想象，如果我们距离真正的最小值只有 0.5 米，但是采用 2 米的步长，那么由于向最小值的方向走的每一步都超过了最小值，我们就会错过最小值。如果我们调节步长，与梯度的大小成正比例，那么我们接近最小值时，斜率也变得平缓了。对于大多数光滑的连续函数，这个假设是合理的。但是对于有时突然一跃而起、有时突然急剧下降的锯齿函数而言，也就是说存在数学家所说的间断点，这不是一个合理的假设。

# 准备数据

在本节中，我们需要思考如何最好地准备训练数据，初始随机权重，甚至设计输出值，给训练过程一个成功的机会。

你没有看错！并不是所有使用神经网络的尝试都能成功，这有许多原因。一些问题可以通过改变训练数据、初始权重、设计良好的输出方案来解决。让我们逐个讨论。

# 输入

仔细观察下图的 S 激活函数。你可以发现，如果输入变大，激活函数就会变得非常平坦。

由于我们使用梯度学习新的权重，因此一个平坦的激活函数会出问题。

回头仔细观察关于权重变化的表达式。权重的改变取决于激活函数的梯度。小梯度意味着限制神经网络的学习能力。这就是所谓的饱和神经网络。这意味着，我们应该尽量保持小的输入。

有趣的是，这个表达式也取决于输入信号 ( ojo_joj​ ) ，因此，我们也不应该让输入信号太小。当计算机处理非常小或非常大的数字时，可能会丧失精度，因此，使用非常小的值也会出现问题。

一个好的建议是重新调整输入值，将其范围控制在 0.0 到 1.0 之间。输入 0 会将 ojo_joj​ 设置为 0 ，这样权重更新表达式将会等于 0 ，从而造成学习能力的丧失，因此在某些情况下，我们会将此输入加上一个小小的偏移，如 0.01 ，避免输入 0 带来麻烦。

# 输出

神经网络的输出是最后一层节点弹出的信号。如果我们使用的激活函数不能生成大于 1 的值，那么尝试将训练目标值设置为比较大的值就有点蠢了。请记住，逻辑函数甚至不能取到 1.0 ，只能接近 1.0 。数学家称之为渐近于 1.0 。

如果我们将目标值设置在这些不可能达到的范围，训练网络将会驱使更大的权重，以获得越来越大的输出，而这些输出实际上是不可能有激活函数生成的。这使得网络饱和，因此我们直到这种情况是很糟糕的。

因此，我们应该重新调整目标值，匹配激活函数的可能输出，注意避开激活函数不可能达到的值。

虽然，常见的使用范围为 0.0 ~ 1.0 ，但是由于 0.0 和 1.0 这两个数也不可能是目标值，并且有驱使产生过大的权重的风险，因此一些人也使用 0.01 ~ 0.99 的范围。

# 随机初始权重

与输入和输出一样，同样的道理也适用于初始权重的设置。由于大的初始权重会造成大的信号传递给激活函数，导数网络饱和，从而降低网络学习到更好的权重的能力，因此应该避免大的初始权重值。

我们可以从 -1.0 ~ +1.0 之间随机均匀地选择初始权重。比起使用非常大的范围，比如说 -1000 ~ +1000 ，这是一个好得多的思路。

我们能够做得更好吗？也许吧。

对于给定特定形状的网络以及特定的激活函数，数学家和计算机科学家曾进行过数学计算，制定出了经验法则，设置了随机初始权重。这是非常 “特定的”！无论如何，让我们继续前进吧。

在此处，我们不纠结于计算细节，但是，其核心思想是，如果很多信号进入一个节点（这也是在神经网络中出现的情况），并且这些信号的表现已经不错了，不会太大，也不会分布得奇奇怪怪，那么在对这些信号进行组合并应用激活函数时，权重应该支持保持这些表现良好的信号。换句话说，我们不希望权重破坏了精心调整输入信号的努力。数学家所得到的经验规律是，我们可以在一个节点传入链接数量的平方根倒数的大致范围内随机采样，初始化权重。因此，如果每个节点具有 3 条传入链接，那么初始权重的范围应该从 −13-\frac{1}{\sqrt{3}}−3​1​ 到 +13+\frac{1}{\sqrt{3}}+3​1​ ，即 ±0.577\plusmn 0.577±0.577 之间。如果每个节点具有 100 条传入链接，那么权重的范围应该在 −1100-\frac{1}{\sqrt{100}}−100​1​ 到 +1100+\frac{1}{\sqrt{100}}+100​1​ ，即 ±0.1\plusmn 0.1±0.1 之间。

直觉上说，这是有意义的。一些过大的初始权重将会在偏置方向上偏执激活函数，非常大的权重将会使激活函数饱和。一个节点的传入链接越多，就有越多的信号被叠加在一起。因此，如果链接更多，那么减小权重的范围，这个经验法则是有道理的。

如果你已经熟悉从概率分布中进行采样的思想，那么这一经验法则实际上讲的是，从均值为 0 、标准方差等于节点传入链接数量平方根导数的正态分布中进行采用。但是，由于经验法则所假设的一些事情，如可替代的激活函数 tanh()tanh()tanh() 、输入信号的特定分布等，可能不是真的，因此，我们不必太担心要精确正确地理解这个法则。

不管你做什么，禁止将初始权重设定为相同的恒定值，特别是禁止将初始权重值设定为 0 。要不然，事情会变得很糟糕。

如果这样做，那么在网络中的每个节点都将接收到相同的信号值，每个输出节点的输出值也是相同的，在这种情况下，如果我们在网络中通过反向传播误差更新权重，误差必定得到平分。你还记得误差按权重比例进行分割吧！那么，这将导致同等量的权重更新，再次出现另一组值相等的权重。由于正确训练的网络应该具有不等的权重（对于几乎所有的问题，这是极有可能的情况），那么由于这种对称性，你将永远得不到这种网络，因此这是一种很糟糕的情况。

由于 0 权重，输入信号归零，取决于输入信号的权重更新函数也因此归零，这种情况更糟糕。网络完全丧失了更新权重的能力。

你还可以做许多其他的事情，对如何准备输入数据、设置权重、组织所需的输出进行优化。以上的思想很容易理解，也可以得到一个相当好的效果，因此，我们就此打住。

如果输入、输出和初始权重数据的准备与网络设计和实际求解的问题不匹配，那么神经网络并不能很好地工作。

一个常见的问题就是饱和。在这个时候，大信号（这有时是由大权重带来的）导致了应用在信号上的激活函数的斜率变得非常平缓。这降低了神经网络学习到更好权重的能力。

另一个问题是零值信号或零值权重。这也可以使网络丧失学习更好权重的能力。

内部链接的权重应该是随机的，值较小，但要避免零值。如果节点的传入链接较多，有一些人会使用相对复杂的规则，如减小这些权重的大小。

输入应该调整到较小值，但不能为零。一个常见的范围为 0.01 ~ 0.99 ，或 -1.0 ~ 1.0 ，使用哪个范围，取决于是否匹配了问题。

输出应该在激活函数能够产生的值的范围内。逻辑 S 函数是不可能生成小于等于 0 或大于等于 1 的值。将训练目标值设置在有效的范围之外，将会驱使产生越来越大的权重，导致网络饱和。一个合适的范围为 0.01 ~ 0.99。
