# 结构冒险

# 数据冒险

写后读 RAW：
解决方式：

  1. 编译器：调换指令顺序
  2. 转发：ALU 转发结果

写回阶段：先写后读
译码阶段：先读后写

1. IF - ID - EX - MEM -  WB
2.      IF - ID -  EX - MEM - WB
3.           IF -  ID -  EX - MEM - WB
4.                 IF -  ID -  EX -  MEM - WB

1.4.    不会冲突，寄存器自转发，不用额外转发。
1.2.3.  冲突，转发。[1]ALU输出转发[2/3]ALU输入

取字指令：
解决方式：
空指令隔开 1 周期，然后转发 ALU

1. IF - ID - EX - MEM -  WB
2.      IF - ID -  EX - MEM - WB
3.           IF -  ID -  EX - MEM - WB


1.2. 转发无法解决 [1]MEM/WB 无法转发到 [2]EX/MEM
1.3. 转发可以解决 [1]MEM/WB 可以转发到 [3]EX/MEM

解决方式：流水线阻塞，阻塞2.

1. IF - ID - EX - MEM - WB
#.      IF - ID -  [] - [] - []
2.           IF -  ID - EX - MEM - WB
3.                 IF - ID -  EX - MEM - WB

# 控制冒险

跳转指令后是否执行
ALU 是否有 0 标志判断是否跳转
解决方法：

  1. 假定分支不发生
  2. 缩短分支延迟（计算目标地址、判断分支条件）
  3. 预测分支结果
  4. 编译器调整指令顺序（延迟时间片）

# 流水线冒险处理

前面寄存器写，后面寄存器读

写：
R 指令 - 写 rd
lw - 写 rt
读：
R 指令 - 读 rs,rt
lw - 读 rs
sw - 读 rs, rt
beq - 读 rs, rt

R 指令在第三个阶段产生结果
lw 在第四阶段产生数据

例题 1

I1        lw    $9, 4($8)           #写9
I2        addu  $11, $10, $9        #写11，读9。有可能与I1数据相关
I3        sw    $12, 12($8)         #不会写
I4        lw    $13, 8($8)          #写13
I5        addu  $14, $10, $13       #写14，读13。可能与I4数据相关
I6        sw    $15, -4($8)         #不会写

画时钟周期图：
无转发时：读取 [I1] WB，无法写入 [I2] ID，产生冒险。

I1        IF - ID -  EX  - MEM - [WB]
I2             IF - [ID] -  EX - MEM - WB

延迟 2 个周期：[I1] WB，可以写入 [I2] ID（加两个空指令）

I1        IF - ID - EX - MEM - [WB]
I2                        IF - [ID] - EX - MEM - WB

编译器指令调度优化：

I1        lw    $9, 4($8)
I4        lw    $13, 8($8)
I3        sw    $12, 12($8)
I2        addu  $11, $10, $9
I5        addu  $14, $10, $13
I6        sw    $15, -4($8)

m 级流水线，n 条指令:
周期数：m+(n-1)+[nop]

无转发：                   5+(6-1)+[2+2] = 14
编译器指令调度优化：        5+(6-1) = 10

例题 2：
有转发时：EX/MEM 转发 ID/EX，即访存 MEM 后 / 执行 EX 后转发执行 EX 前

I1        lw        $6, -16($6)       #写6
I2        sw        $6, 10(6)         #     读6。I1
I3        add       $6, $2, $6        #写6，读6。I1
I4        sub       $8, $9, $6        #写8，读6。I3
I5        or        $6, $11, $12      #写6

充分转发：[I1] MEM 后转发 [I2] EX 前，加 1 个 NOP，[I3] 同时解决

IF - ID - EX - [MEM] - WB
          IF -  [ID] - EX - MEM - WB

lw -> nop -> sw -> add -> ...

充分转发：[I3] EX 后转发 [I4] EX 前，不需要加 NOP

IF - ID - EX - [MEM] - WB
     IF - ID - [EX] - MEM - WB

... -> add -> sub - > or

总结：
在无转发情况下，lw+add 后，加 2 个 NOP。
在完美转发情况下，lw 后加 1 个 NOP，R 型指令后不加 NOP。
