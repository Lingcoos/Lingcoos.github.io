# 数据结构 (Data Structures)

常见的数据结构包括数组、链表、栈、队列、哈希表、树、堆、图。

# 逻辑结构 (Logic Data Structures)

  • 逻辑结构揭示了数据元素之间的逻辑关系。
    • 逻辑结构包含线性结构和非线性结构。

# 线性结构 (Linear Data Structures)

  • 线性结构指数据在逻辑关系上呈线性排列。

例如：数组、链表、栈、队列、哈希表。

# 非线性结构 (Non-linear Data Structures)

  • 非线性结构指数据在逻辑关系上呈非线性排列。

例如：树、堆、图、哈希表。

  • 非线性结构包含树状结构和网状结构。

线性结构：数组、链表、队列、栈、哈希表，元素之间是一对一的顺序关系。
树形结构：树、堆、哈希表，元素之间是一对多的关系。
网状结构：图，元素之间是多对多的关系。

# 物理结构 (Physical Data Structures)

当算法程序运行时，正在处理的数据主要存储在内存中。系统通过内存地址来访问目标位置的数据。

  • 物理结构反映了数据在计算机内存中的存储方式，可分为:
    • 连续空间存储 (数组)
    • 分散空间存储 (链表)

所有数据结构都是基于数组、链表或二者的组合实现的。

# 复杂度 (Complexity)

时间复杂度和空间复杂度是衡量一个算法效率的重要标准。

# 时间复杂度 (Time Complexity)

我们用时间复杂度来衡量一个算法所花费的时间随数据规模而增长的趋势。

# 数组和链表 (Array and Linked List)

# 数组 (Array)

数组 (Array) 是一种线性数据结构，其将相同类型的元素存储在连续的内存空间中。
我们将元素在数组中的位置称为该元素的索引 (index) 。

优点：查找数据。时间复杂度：O(1)O(1)O(1) 。
缺点：插入数据、删除数据。时间复杂度：O(n)O(n)O(n) 。

丢失元素：由于数组的长度不可变，因此在插入元素后，超出数组长度范围的元素会丢失。

内存浪费：我们可以初始化一个比较长的数组，只用前面一部分，这样在插入数据时，丢失的末尾元素都是 “无意义” 的，但这样做会造成部分内存空间浪费。

# 链表 (Linked List)

链表 (Linked list) 是一种线性数据结构，其中的每个元素都是一个节点 (Node) 对象，各个节点通过指针 (Pointer) 相连接。指针指向下一个节点的内存地址，通过它可以从当前节点访问到下一个节点。

优点：插入数据、删除数据。时间复杂度： O(1)O(1)O(1) 。
缺点：查找数据。时间复杂度： O(n)O(n)O(n) 。

# 单向链表 (Singly Linked List)

单向链表 (Singly Linked List) 包含两个域，信息域和指针域，指针指向下一个节点，最后一个节点则指向空值 (Null) 。

# 树 (Tree)

树 (Tree) 是 N(N>=0)N(N>=0)N(N>=0) 个结点的有限集合， N=0N=0N=0 时称为空树。

根节点 (Root Node) ：位于二叉树顶层的节点，没有父节点。
叶子节点 (Leaf Node) ：没有子节点的节点，其两个指针均指向空。
边 (Edge) ：连接两个节点的线段，即指针。
节点所在的层 (Level) ：从顶至底递增，根节点所在层为 1。
节点的度 (Degree) ：节点的子节点的数量。在二叉树中，度的取值范围是 0、1、2。
二叉树的高度 (Height) ：从根节点到最远叶子节点所经过的边的数量。
节点的深度 (Depth) ：从根节点到该节点所经过的边的数量。根节点的深度为 0。
节点的高度 (height) ：从距离该节点最远的叶子节点到该节点所经过的边的数量。

  • 树的基本性质：
    • 节点数=总度数+1节点数 = 总度数 + 1节点数=总度数+1

# 完美二叉树 (Perfect Binary Tree)

A Perfect Binary Tree(PBT) is a tree with all leaf nodes at the same depth. All internal nodes have degree 2.

一个深度为k(k>=−1)k(k>=-1)k(k>=−1) 且有2k+1−12^{k+1}-12k+1−1 个结点的二叉树称为完美二叉树 (Perfect Binary Tree) 。

国内的数据结构教材大多翻译为 "满二叉树"

# 完全二叉树 (Complete Binary Tree)

A Complete Binary Tree (CBT) is a binary tree in which every level, except possibly the last, is completely filled, and all nodes are as far left as possible.

除了最后一层可以不完全填充，其他每一层都填满且其叶子结点都靠左对齐的二叉树称为完全二叉树 (Complete Binary Tree) 。

# 满二叉树 (Full Binary Tree)

A Full Binary Tree (FBT) is a tree in which every node other than the leaves has two children.

所有非叶子结点的度都是 2 的二叉树称为满二叉树 (Full Binary Tree)

# 平衡二叉树 (Balanced Binary Tree)

平衡二叉树 (Balanced Binary Tree) ，又称 AVL 树，是一种特殊的二叉搜索树 (Binary Search Tree) 。

对于 AVL 树中的每一个结点，他的左子树和右子树的高度之差(平衡因子)的绝对值不超过 1。且左子树和右子树都是一棵平衡二叉树，满足递归的定义。

# 删除节点

  • 如果要删除的结点是叶子结点，那么直接删除即可。
  • 如果要删除的结点只有左子树或者右子树，那么将其删除，并让它的左或者右子树代替它的位置。
  • 如果要删除的结点既有左子树也有右子树，那么需要让它直接前驱 (直接后继) 替换它的值，并且删除它的直接前驱或者直接后继。 (因为它的直接前驱必然是它的左子树的最右结点，必然没有右子树，所以转化为第 1 或者第 2 种情况，直接后继同理)

可以通过中序遍历来找到该节点 (左节点 <<< 根节点 <<<右节点) 。

# *AVL 树 (AVL Tree)

AVL 树既是二叉搜索树也是平衡二叉树，同时满足这两类二叉树的所有性质，因此也被称为平衡二叉搜索树 (Balanced Binary Search Tree) 。

# 平衡因子 (Balanced Factor)

平衡二叉树（AVL 树）上某节点的平衡因子 (Balanced Factor) 定义为该节点的左子树的高度减去右子树的高度。

BF(node)=h(node.left())−h(node.right())BF(node) = h(node.left()) - h(node.right())BF(node)=h(node.left())−h(node.right())

平衡二叉树（AVL 树）上所有节点的平衡因子只可能为 -1、0、1。

只要二叉树上有一个节点的平衡因子的绝对值大于 1，该二叉树就不是平衡二叉树（AVL 树）。

  • 图数据结构包含以下常用术语:

    • 邻接 (adjacency) ：当两顶点之间存在边相连时，称这两顶点 “邻接”。

# 邻接矩阵 (Adjacency Matrix)

邻接矩阵 (Adjacency Matrix) 使用一个 n∗nn*nn∗n 大小的矩阵来表示图，每一行（列）代表一个顶点，矩阵元素代表边，用 111 或 000 表示两个顶点之间是否存在边。

  • 邻接矩阵具有以下特性:
    • 顶点不能与自身相连，因此邻接矩阵主对角线元素没有意义。
    • 对于无向图，两个方向的边等价，此时邻接矩阵关于主对角线对称。
    • 将邻接矩阵的元素从 111 和 000 替换为权重，则可表示有权图。

# 图的遍历

  • 图的遍历可能会遇到以下问题：
    • 图的某一顶点可能与其他多个顶点邻接，因此可能出现重复访问的情况，即存在回路。
    • 若图不是连通的，从起点出发可能到不了所有的顶点。
  • 解决方法
    • 设置全局数组 visited [] ，来标记已访问过的顶点。
    • 其初始值为 false 。
    • 一旦某个顶点 iii 被访问，则 visit [i] 的值变为 true 。
    • 通过检查 visit [n] 是否都为 true 来判断遍历是否结束。
    • 若存在某个顶点未被访问，则可以从某个未被标记的顶点开始继续遍历。

图的遍历的实质就是查找每个顶点的邻接点的过程。
